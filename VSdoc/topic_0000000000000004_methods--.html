The following tables list the members exposed by the Procedure type.

Methods
Name Description
Clone Creates a deep copy of the procedure.
ContainsParameter Checks if the parameter is in the collection.
ExecuteJson For *Returns Json* procedures where the raw Json result is the desired return type.
ExecuteJson<T> For *Returns Json* procedures where an object of T is the desired return type.
ExecuteJsonList<T> For *Returns Json* procedures where a List<T> is the desired return type.
ExecuteNonQuery Executes procedure with no results. Sets the output parameters and return value.
ExecuteReader<T> For result set procedures where an object of T is the desired return type. Uses implicit or explicit mapping.
ExecuteReaderAsJson<T> Utilizes ExecuteReader and serializes result to Json.
ExecuteReaderDataTable For result set procedures where the desired return type is a Data Table.
ExecuteReaderList<T> For result set procedures where a List of object of T is the desired return type. Uses implicit or explicit mapping.
ExecuteReaderListAsJson<T> Utilizes ExecuteReaderList serializes result to Json.
GetParameter Finds a parameter by any of the name variations.
GetValue<T> Gets the parameter value as T based on the parameter name.
LoadFromJson Sets the parameters values from Json. Note that when the Json provides a list the parameter must have a user defined table type as the parameter.
LoadFromObject<T> Utilizes ObjectPropertyToParameterNameMapping to set the values of the parameters in the procedure
LoadFromQuery Sets the parameter values from the query parameters. Note that multiple values (s=1 s=2 s=3) that evaluate to a list are not supported. For multiple records use the LoadFromJson method.
ReturnValue<T> Short cut to get parameter named RETURN_VALUE.
SetValue<T> Sets the value as T based on the name.
